## Why games?

Not because learning should be "fun."
Because **good game design and good pedagogy are the same thing.**

A well-designed game level teaches you exactly one thing —
through **failure, repetition, feedback, and mastery** —
in the exact order you need to learn it.

That's how humans actually learn.

</div>

---

<br />

## The insight behind ahafrog

ahafrog doesn't bolt games onto content.
playing the game *is* understanding the concept.
Not a metaphor. Not an analogy. The same cognitive act.

---

<br />

## What you can do with it

<br />

**📄 &nbsp; Upload anything. Get a playable course.**

Textbooks. Research papers. Internal documentation. Product specs. Training manuals. API references. Lecture notes. Wikipedia articles.

Paste the source material → ahafrog's AI reads it, identifies the core concepts, maps each one to its natural game mechanic, and generates a series of playable micro-lessons — complete with visuals, interactive gameplay, and adaptive pacing.

No game design experience. No curriculum design background. No engineering required.

<br />

**🎯 &nbsp; Every concept gets the game it deserves.**

A lesson on sorting algorithms becomes a racing game where you sort under pressure.
A module on logical fallacies becomes a trap-spotting challenge.
A chapter on chemical reactions becomes a combination puzzle.
A section on negotiation tactics becomes a live tradeoff scenario.

The game *is* the lesson. The mechanic *is* the mental model.

<br />

**🔁 &nbsp; Learning that compounds.**

ahafrog tracks exactly what you know and when you're about to forget it — then brings that concept back as a game at the perfect moment. Each replay reinforces the same mental model through a slightly different challenge, deepening understanding instead of just refreshing memory.

<br />

**🌍 &nbsp; Any subject. Any language. Any audience.**

English · 中文 · 日本語 · Español · Français · 한국어 · العربية · Deutsch

From elementary math to graduate-level theory. From onboarding new hires to mastering a second language. If it can be written down, it can be turned into a game on ahafrog.

<br />

**⚡ &nbsp; Instant. Everywhere. Always.**

Opens in any browser on any device in under a second — anywhere in the world. No installs. No downloads. No friction between a learner and the thing they're trying to learn.

## 为什么是游戏？

不是因为学习应该"有趣"。
而是因为**好的游戏设计和好的教学法，本质上是同一回事。**

一个设计精良的游戏关卡，只教你一件事——
通过**失败、重复、反馈和掌握**——
以你真正需要的顺序。

这才是人类真正的学习方式。

</div>

<br />

## ahafrog 的核心洞察

每一个学习目标，都对应一种游戏机制。
结果是：玩游戏的过程，就是理解概念的过程。

## 你能用它做什么

<br />

**📄 &nbsp; 上传任何材料，获得可玩课程。**

教科书、研究论文、内部文档、产品规格书、培训手册、API 文档、课堂笔记、维基百科文章。

粘贴原始材料 → ahafrog 的 AI 阅读并识别核心概念，将每个概念匹配到最适合的游戏机制，自动生成一系列可玩的微型课程——配有视觉效果、互动玩法和自适应节奏。

无需游戏设计经验。无需课程设计背景。无需任何工程技能。

<br />

**🎯 &nbsp; 每个概念都得到它该有的游戏。**

排序算法的课程，变成在压力下竞速排序的比赛。
逻辑谬误的模块，变成识破陷阱的挑战。
化学反应的章节，变成组合配对的谜题。
谈判策略的段落，变成实时权衡取舍的场景。

游戏就是课程本身。机制就是心智模型本身。

<br />

**🔁 &nbsp; 持续复利的学习。**

ahafrog 精确追踪你知道什么、何时会遗忘——然后在最合适的时机，以游戏的方式把那个概念重新带回来。每次回放都通过略有不同的挑战强化同一个心智模型，加深理解，而不只是刷新记忆。

<br />

**🌍 &nbsp; 任何学科，任何语言，任何受众。**

English · 中文 · 日本語 · Español · Français · 한국어 · العربية · Deutsch

从小学数学到研究生理论。从新员工入职到掌握第二语言。只要能写下来，就能在 ahafrog 上变成游戏。

<br />

**⚡ &nbsp; 即开即用，全球可达。**

任何设备、任何浏览器，一秒内打开——无需安装，无需下载，学习者和他们想学的东西之间，没有任何摩擦。
